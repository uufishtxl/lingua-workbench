# 项目经验

1. 遇到的核心挑战 (The Challenge)： 当时产线的业务逻辑远比预想的复杂。厂端要求我们将系统与现有的 BOM (物料清单) 系统打通。 但在与产线工程师深度访谈后，我发现了一个致命问题：BOM 数据是‘静态’的，而产线动作是‘动态’的。 现有的 BOM 系统只告诉我有‘什么零件’，但没告诉我‘怎么装配’。这就导致我们无法直接计算工时和模拟动作。

2. 我的解决方案 (The Solution)： 为了解决这个问题，我利用 Python 设计了一套中间层的数据逻辑：

第一步：跨系统数据融合（Data Fusion）。 我发现 IE 工程师使用的 DFC (Design for Cost) 系统里包含成本计算属性。于是我写脚本将 DFC 中的工艺属性（如螺丝规格、卡扣类型）清洗并合并到 BOM 的基础数据上。

第二步：引入 DSL (领域特定语言) 思维。 既然现有的数据结构不支持，我就自己定义。我参考 DSL 的设计逻辑，设计了一套映射规则。

第三步：动态对象生成。 我编写了 Python 核心算法，让程序能够读取融合后的数据，动态生成‘动作描述对象’。 比如，系统读到‘4个螺丝+十字接口’的数据对象，就会自动生成‘取螺丝 -> 对准 -> 锁附’的标准动作序列。

3. 最终结果 (The Result)： 通过这套逻辑，我们成功把死板的物料数据，转化为了带有工时属性（MODAPTS）的可计算数据。这不仅解决了系统对接问题，还为后来的 AI 模拟打下了数据基础。”

# Debug 问题

在公司的 POC 项目中，因为主要是快速验证可行性，更多是架构设计上的挑战。 但在我的深度自研项目（那个英语学习平台）中，因为涉及到底层 DOM 操作和复杂的安全鉴权，我遇到过几个非常有意思的技术难题。

在开发音频笔记的高亮功能时，我遇到了一个极难定位的 Bug。 【现象】 我使用原生 DOM 的 Range 和 Selection API 来计算选区位置，但在某些特定情况下，计算出的偏移量总是错位的，导致高亮错位。 【排查】 我打了无数断点，最后发现 DOM 结构里多了一些莫名其妙的注释节点 ``。 【根因】 原来是 Vue 的 v-if 指令。为了保证重新渲染的效率，Vue 在条件为 false 时，会在 DOM 原地留下一个注释节点作为‘占位符’（Placeholder）。而原生的 DOM API 在计算 childNodes 索引时，会把这个注释节点也算进去，导致索引偏移。 【解决】 查明原因后，我将 v-if 换成了 v-show。因为 v-show 只是切换 display: none，DOM 结构保持不变，不会插入额外的节点，从而完美解决了偏移问题。

另一个是在做 Django 后端鉴权时。我使用的是 SimpleJWT 库，但在前端刷新 Token 时总是失败。 【排查】 我检查了后端返回的 Response Body，发现里面死活没有 refresh token。排查了很久文档和源码配置。 【根因】 后来发现这是 Django 为了安全（防止 XSS 攻击），默认配置把 Refresh Token 放在了 HttpOnly Cookie 里，而不是放在响应体（JSON）里。这是为了更安全，但也导致前端无法直接通过 JS 读取。 【解决】 理解了这个安全机制后，我修改了前端的请求逻辑，配置了 credentials: include 让请求自动带上 Cookie，并在后端调整了中间件逻辑，最终打通了安全的刷新流。

我的 Debug 风格是**‘假设-验证-记录’闭环。 我拥有自己的一套 AI 辅助工作流。当遇到复杂报错时，我会将错误栈和上下文喂给 AI 进行初步分析。 但我发现 AI 经常会陷入‘幻觉死循环’。这时候我的经验就发挥作用了：我会重置会话**，或者调整 Prompt 引导它去排查特定的底层原理（比如刚才的 Vue 占位符）。 最重要的是复盘： 问题解决后，我会让 AI 帮我生成一份结构化的技术笔记，归档到我的 GitHub 知识库里。 这让我以后遇到类似问题能秒解，这也是我**‘个人知识库 Agent’**的一部分。

# 前端

这是一个从 Adobe Flash 迁移到现代 Web 的重构项目。 难点在于，老系统依赖 Flash 强大的动画引擎来实现多动画、高交互的硬件模拟。我接手后发现，市面上现有的 UI 库（如 ElementPlus）都是为 B 端表单设计的，根本无法实现这种像素级的硬件情景重现（比如开机动画、视图切换、亮度调节的实时反馈）。

所以，我做了一个大胆的技术决策：抛弃现成 UI 库，手写一套定制化的组件系统。

组件架构设计： 我深入研究了 ElementPlus 等开源库的底层逻辑，设计了一套自己的组件通信机制（Props/Emit/Provide/Inject），实现了复杂的父子组件和跨层级状态同步。

CSS 攻坚： 我利用 CSS3 和 Vue Transition 完美复刻了 Flash 级别的平滑动画效果，还原了显示器硬件的真实操作手感。

【结果】 最终我交付了一套纯 Vue 3 + Tailwind 的高性能模拟器，解决了 Flash 停更的安全隐患，这个项目打磨了我非常扎实的 CSS 功底 和 组件化思维。
